### 2. 각 Feature별 데이터 요구사항 (Data Requirements)

각 기능이 정상적으로 작동하기 위해 시스템이 요구하는 데이터의 종류와 형태를 정의합니다.

#### ⚙️ F1: 실시간 데이터 수집 모듈 요구사항
*   **데이터 유형:** 현재가(시가, 고가, 저가, 종가), 거래량, 체결 시간.
*   **요구 사항:** 키움 RestApi에서 제공하는 **실시간 시세 스트림**에 대한 안정적인 연결 및 오류 처리 메커니즘.
*   **데이터 형식:** Timestamp (UTC/KST), Security Code, Price (Float), Volume (Integer).

#### ⚙️ F2: 기술적 분석 지표 계산 모듈 요구사항
*   **데이터 유형:** 시계열 데이터 (최소 N일 또는 N봉의 종가 및 거래량).
*   **요구 사항:** 과거 데이터를 저장할 수 있는 **시계열 데이터베이스(Time-Series DB)** 구조 설계.
*   **필수 지표 (예시):**
    *   **이동평균선 (MA):** 단기(5일, 20일), 중기(60일) 등 다양한 기간 설정 가능해야 함.
    *   **상대강도지수 (RSI):** 특정 기간(14일 기준) 계산 및 범위 정의 필요.
    *   **이동평균 수렴/확산 지표 (MACD):** MACD 선, 시그널 선, 히스토그램 값 저장 구조 필요.

#### ⚙️ F3: 상승/하락 추세 분류 모듈 요구사항
*   **데이터 유형:** F2에서 계산된 모든 기술적 지표의 **상대적인 변화율**.
*   **요구 사항:** 다중 지표 간의 **상관관계 분석 프레임워크**가 필요하며, 이를 통해 단순한 단일 지표 기반이 아닌 복합적인 추세 판단 로직을 구현해야 함.
*   **데이터 형식:** 추세 분류 결과 (Categorical: 상승/하락/횡보) 및 해당 분류의 신뢰도 점수.

#### ⚙️ F4: 급등/강성주 필터링 모듈 요구사항
*   **데이터 유형:** 실시간 데이터(F1)와 지표 변화율(F2, F3)의 **동시 비교**.
*   **요구 사항:** '급등'을 정의하기 위한 **거래량 대비 가격 변화율 임계값 (Threshold)** 및 **변동성 측정 지표 (Volatility Index)**의 데이터 요구.
*   **데이터 형식:** 필터링된 종목 목록, 각 종목별 급등/강성 점수.

## F1/F2 구현을 위한 Time-Series DB 구조 설계안 및 데이터 요구사항 정의

본 문서는 키움 RestApi 기반 주식 검색 알고리즘 시스템(상승예측, 급등예측, 현재강성주 파악) 구현에 필요한 실시간 데이터 수집(F1) 및 기술적 분석 지표 계산(F2) 모듈을 지원하기 위한 Time-Series Database (TSDB)의 구조 설계안과 각 Feature Set별 데이터 요구사항을 정의합니다.

---

### 1. Time-Series DB (TSDB) 구조 설계안

**목표:** 실시간 시계열 데이터의 효율적인 저장, 고속 인덱싱 및 복잡한 기술적 분석 계산 지원.

#### 1.1. 핵심 데이터 모델 (Schema Definition)

| 필드명 | 데이터 타입 | 설명 | 용도 | 인덱스/파티션 전략 |
| :--- | :--- | :--- | :--- | :--- |
| **Timestamp** | DateTime (UTC) | 데이터가 기록된 정확한 시점 (필수 Key) | 시계열 정렬 및 시간 기반 검색 | Primary Index, Time-based Partitioning |
| **Symbol** | String | 종목 코드 (예: '005930') | 종목별 분리 및 그룹화 | Secondary Index |
| **Data_Type** | Enum/String | 데이터 유형 (예: 'OHLCV', 'Indicator', 'Volume') | 데이터의 종류 구분 | Partitioning Key |
| **Value** | Float / Decimal | 실제 측정된 값 (가격, 거래량, 지표 값 등) | 핵심 시계열 데이터 | - |
| **Source** | String | 데이터 출처 (예: 'Kiwoom_Realtime', 'Calculated_EMA') | 데이터 신뢰성 및 추적 | - |

#### 1.2. 저장 구조 상세 (Storage Structure Detail)

TSDB는 **종목별 파티션(Partitioning by Symbol)**을 기본으로 하여 데이터를 분리하고, 각 종목 내에서는 **시간 기반 인덱싱**을 적용합니다.

*   **파티셔닝 전략:** `Symbol`을 기준으로 물리적 또는 논리적 파티션 설정. (데이터 양이 방대해질 경우, 시간 범위별 추가 파티션 고려)
*   **인덱싱 전략:** `(Symbol, Timestamp)` 조합에 대해 B-Tree 또는 LSM Tree 기반의 인덱스를 사용하여 특정 종목의 과거 데이터 조회 속도를 극대화합니다.

#### 1.3. F1 (실시간 데이터 수집 모듈) 지원 구조

F1은 주로 **스트리밍 데이터**를 처리하므로, DB는 높은 쓰기(Write) 성능을 지원해야 합니다.
*   **구조 초점:** Ingestion Pipeline 최적화. 데이터가 도착하는 즉시 타임스탬프와 함께 임시 저장 후 최종 구조에 병합하는 방식 채택.

#### 1.4. F2 (기술적 분석 지표 계산 모듈) 지원 구조

F2는 **과거 데이터 기반의 복잡한 연산**을 요구하므로, 읽기(Read) 성능과 데이터 무결성이 최우선입니다.
*   **구조 초점:** 집계된 지표(OHLCV 외)를 미리 계산하여 저장하는 캐시 레이어(Materialized View)를 함께 구성합니다.
    *   예: 일봉, 주봉 데이터는 원본 OHLCV 외에 최종적으로 계산된 이동평균선(EMA, SMA), RSI, MACD 등을 별도의 테이블로 저장하여 실시간 분석 요청 시 JOIN 없이 즉시 응답 가능하게 설계합니다.

---

### 2. Feature Set별 데이터 요구사항 정의 (Data Requirements)

F1과 F2를 성공적으로 구현하기 위해 각 모듈이 요구하는 최소 및 확장 데이터 세트를 명확히 정의합니다.

from typing import Dict, Any
from datetime import datetime

class StockTimeSeriesData:
    """
    주식 시계열 데이터를 저장하고 관리하기 위한 핵심 데이터 모델.
    TSDB 설계안(Symbol별 파티셔닝, 시간 인덱싱)을 반영하여 설계함.
    """
    def __init__(self, symbol: str):
        # 1. 기본 식별자 (Primary Indexing Key)
        self.symbol: str = symbol  # 종목 코드 (예: '005930')

        # 2. 메타데이터
        self.data_type: str = "OHLCV"  # 데이터 유형 (OHLCV, Indicator 등)
        self.source: str = "Kiwoom_Realtime"  # 데이터 출처

        # 3. 시계열 데이터 저장소 (시간 기반 인덱싱의 핵심)
        # 실제 데이터는 리스트 형태로 저장하여 Pandas DataFrame으로 변환 용이하게 함.
        # 구조: List[Tuple[datetime, Value]] 또는 Dict
        self.time_series: Dict[str, Any] = {}

    def add_data_point(self, timestamp: datetime, value: float, data_type: str = "OHLCV"):
        """
        새로운 시계열 데이터 포인트를 추가합니다.
        시간 기반 인덱싱을 위해 타임스탬프를 키로 사용합니다.
        """
        if self.data_type != data_type:
            # 데이터 타입이 다르면 새로운 객체를 생성하거나 예외 처리 필요 (F2 단계에서 분리 예정)
            print(f"Warning: Data type mismatch for {self.symbol}. Storing as {self.data_type}.")

        key = timestamp.isoformat()
        
        if key not in self.time_series:
            self.time_series[key] = {
                "timestamp": timestamp,
                "value": value,
                "type": data_type,
                "source": self.source
            }
        else:
            # 이미 데이터가 존재하는 경우 덮어쓰기 또는 병합 로직 필요 (실시간에서는 덮어쓰기 가정)
            self.time_series[key]["value"] = value


    def get_data(self) -> list[Dict[str, Any]]:
        """
        저장된 시계열 데이터를 리스트 형태로 반환합니다.
        Pandas DataFrame으로 변환하기 용이한 구조로 출력합니다.
        """
        results = []
        for key, data in self.time_series.items():
            results.append({
                "Timestamp": data["timestamp"].isoformat(),
                "Value": data["value"],
                "DataType": data["type"],
                "Source": data["source"]
            })
        return results
