#### 3.1. 티어 선택 및 필터 (Tier Selector)

#### 3.2. 핵심 지표 카드 (KPI Cards)

두 가지 핵심 KPI(PTI와 ROI)에 대해 Basic/Pro 티어별 비교를 제공합니다.

**카드 1: 시스템 안전성 지수 (PTI - System Safety Index)**
*   **제목:** PTI Score
*   **시각화:** **게이지 차트 (Gauge Chart)** 또는 **진행 바 (Progress Bar)** 사용.
    *   **Basic Shield:** 목표치 대비 현재 안정성 수준 표시.
    *   **Pro Hedger:** Basic Shield보다 높은 안정성을 시각적으로 표현하며, 안전 마진(Margin)의 차이를 강조.
    *   **색상 적용:** PTI 값이 높을수록 `System Green` 톤으로 변화 (예: 80% 이상은 녹색 계열).

**카드 2: 수익화 KPI (Hedging ROI - Hedging Return on Investment)**
*   **제목:** Hedging ROI
*   **시각화:** **바 차트 (Bar Chart)** 또는 **대시보드 비교 카드**.
    *   **Basic Shield:** 특정 기간(예: 30일)의 평균 ROI를 표시.
    *   **Pro Hedger:** Basic Shield 대비 **ROI 증가율 (%)**을 명확히 표시하여, Pro 티어가 제공하는 추가적인 수익 가치(헤징 효과)를 강조합니다.
    *   **색상 적용:** ROI 값이 긍정적일 경우 `System Green`으로 채우고, 리스크가 발생했을 경우 경고 색상(`Deep Navy Blue`의 어두운 톤)을 사용합니다.

## 1. 디자인 시스템 정의 (Design System Definition)

| 요소 | 값 | 설명 |
| :--- | :--- | :--- |
| **Primary Color** | `Deep Navy Blue` (`#000080` 또는 유사한 짙은 네이비) | 배경, 주요 프레임, 안정적인 데이터 영역에 사용. 신뢰감 강조. |
| **Accent Color** | `System Green` (`#00FF00` 또는 밝은 녹색 계열) | 성공 지표(ROI), 안전성 점수(PTI), 긍정적 상태를 표시. |

## 2. 레이아웃 구조 (Layout Structure)

대시보드는 **사이드바(Navigation)**, **상단 헤더(Summary)**, 그리고 **메인 영역(Detailed Metrics)** 세 부분으로 구성됩니다.

### A. 사이드바 (Navigation - 좌측)
*   **스타일:** Deep Navy Blue 배경에 밝은 회색 텍스트.
*   **구성:**
    1.  로고/브랜드명 (`갓더주식`)
    2.  메뉴 항목: 대시보드, 시스템 현황, 리스크 분석(PTI), 수익성 분석(ROI), 설정 및 티어 관리.

### B. 상단 헤더 (Summary - 최상단)
*   **스타일:** Deep Navy Blue 배경으로 전체 페이지의 안정감을 유지.
*   **구성:** 현재 선택된 티어(`Basic Shield` / `Pro Hedger`)를 명확히 표시하고, 시스템 상태(예: "시스템 안전: 안정", "헤징 ROI: 12%")를 요약하여 보여줍니다.

### C. 메인 영역 (Detailed Metrics - 중앙)
이 영역은 **PTI (안전성)**와 **ROI (수익성)** 두 가지 핵심 KPI를 티어별로 비교하는 데 집중합니다.

## 3. 핵심 시각화 컴포넌트 상세 설계

### 섹션 1: 시스템 안전성 지수 (PTI - Protection & Stability Index)

**목표:** 시스템의 안정성과 리스크 관리 능력을 직관적으로 전달.
**시각화 방식:** 게이지(Gauge) 차트 및 상태 표시등(Status Indicator).

| 티어 | PTI 값 범위 | 시각적 표현 | 색상 적용 | 설명 |
| :--- | :--- | :--- | :--- | :--- |
| **Basic Shield** | 0 ~ 70 | 게이지 바 (회색 계열) | Deep Navy Blue 배경, 경고(Warning) 상태 시 연한 주황색. | 최소 안전장치 확보 상태. |
| **Pro Hedger** | 71 ~ 100 | 게이지 바 (밝은 녹색 계열) | System Green 영역이 채워지며 최대치에 근접. | 최고 수준의 안정성 및 리스크 관리 달성 상태. |

*   **디테일:** 각 티어별로 PTI 수치가 표시되며, 사용자가 특정 지표(예: MDD Limit 준수율)를 클릭하면 해당 데이터가 어떻게 계산되었는지에 대한 간략한 설명이 툴팁으로 제공됩니다.

### 섹션 2: 수익화 KPI (헤징 ROI - Hedging Return on Investment)

**목표:** 시스템 운영을 통한 실제 수익성과 효율성을 강조.
**시각화 방식:** 막대 차트(Bar Chart) 및 변화율 표시.

| 티어 | 헤징 ROI 값 범위 | 시각적 표현 | 색상 적용 | 설명 |
| :--- | :--- | :--- | :--- | :--- |
| **Basic Shield** | 10% ~ 35% | 막대 차트 | Deep Navy Blue 배경, 회색 계열. | 기본적인 헤징 수익 달성 상태. |
| **Pro Hedger** | 36% 이상 | 막대 차트 | System Green 영역이 채워지며 높은 수치 강조. | 최대의 수익성과 효율적인 자산 관리 달성 상태. |

*   **디테일:** ROI 변화 추이를 보여주는 작은 라인 차트를 함께 배치하여, 시간이 지남에 따른 수익성의 흐름을 파악할 수 있게 합니다.

### 섹션 3: 데이터 흐름 및 연동 (Data Flow & Integration)

*   **구성:** 두 KPI(PTI, ROI)의 산출 근거가 되는 핵심 데이터 포인트(예: 현재 MDD 리스크 레벨, 예측 알고리즘 신뢰도 점수 등)를 작은 카드 형태로 나열합니다.
*   **스타일:** 이 섹션은 배경을 아주 연한 회색(`#F0F0F0`)으로 처리하여 메인 KPI 영역과 명확히 구분하며, 모든 데이터는 Deep Navy Blue와 System Green 톤으로 통일하여 데이터 기반 신뢰를 극대화합니다.

def get_dashboard_metrics(user_tier: str) -> Dict[str, Any]:
    """
    선택된 티어에 따른 PTI 및 ROI를 반환합니다. (프론트엔드 요청 처리용)
    """
    print(f"INFO: Calculating metrics for tier: {user_tier}")
    
    # 실제 데이터는 DB 또는 계산 결과에서 로드되어야 함. 여기서는 예시 데이터로 대체합니다.
    if user_tier == "Basic Shield":
        # Basic 티어의 가정된 값 (실제로는 계산 모듈 호출)
        pti = 75.0  # 시스템 안전성 지수

                "PTI": round(pti, 2),
                "ROI": round(roi, 2)
            }
        }
    }

#### 1. 핵심 가설 재확인 (Value Proposition Alignment)

| **PTI** (시스템 안전성 지수) | 예측 알고리즘의 안정성 및 데이터 무결성 측정. | 투자 시스템이 얼마나 신뢰할 수 있는지에 대한 **심리적 안전감** 제공. | **높음**: 이는 사용자가 위험을 인지하고 투자 결정을 내리는 데 필수적인 기반이다. |

#### 2. 예측 알고리즘과의 매핑 검증 (Algorithm-to-KPI Mapping)

시스템이 수학적 논리를 정확히 반영하고 있는지 확인하기 위해, 다음 세 가지 연결 고리가 필수적으로 작동해야 합니다.

1.  **예측 결과 $\rightarrow$ 안전성 지수 (PTI):**
    *   알고리즘의 예측 오차($\text{Error}$)가 $MDD_{limit}$ 및 $SR_{target}$ 경계값에 도달했을 때, 시스템은 즉시 PTI를 하향 조정하거나 `system_safe_mode`를 활성화해야 합니다.
    *   **검증 포인트:** 알고리즘의 미세한 예측 변화(노이즈)가 PTI에 비례하여 반영되어야 하며, 경계값 도달 시 안전장치 트리거는 **즉각적이고 일관적**이어야 합니다.

2.  **안전성 지수 $\rightarrow$ 헤징 ROI:**
    *   PTI가 낮아질수록(위험 증가), 시스템은 자동으로 헤징 모듈의 보수적인 전략을 실행하여 잠재적 손실을 최소화해야 합니다.
    *   **검증 포인트:** PTI와 헤징 전략 간의 피드백 루프가 **역논리적으로 작동**해야 합니다. 즉, 안전성 지수가 낮으면(PTI $\downarrow$), 헤징 ROI는 보수적인 수익 목표($SR_{target}$)를 우선시하는 방향으로 조정되어야 합니다.

3.  **KPI 전달 $\rightarrow$ 프론트엔드 (사용자 경험):**
    *   프론트엔드는 복잡한 수학적 수치 대신, **'안전성 레벨(PTI: Green/Yellow/Red)'**과 **'예상 수익률 변화(ROI %)'**라는 직관적인 시각화로 정보를 제공해야 합니다.
    *   **검증 포인트:** 프론트엔드에 전달되는 데이터는 계산된 PTI와 ROI의 **절대값보다 그 의미(Risk Level, Opportunity Level)**가 더 중요하게 표시되어야 하며, 이는 Designer가 설계한 UI/UX 프로토타입과 완벽히 일치해야 합니다.

CEO의 지시에 따라, 시스템 안전성 지수(PTI)와 수익화 KPI(헤징 ROI)를 시각화하여 '안전성 레벨'을 최우선으로 적용한 최종 대시보드 UI/UX 프로토타입을 완성하겠습니다. 확정된 디자인 가이드라인(`Deep Navy Blue`, `System Green`)과 데이터 기반 신뢰 원칙을 반영하여 구체적인 설계안을 제시합니다.

다음은 최종 대시보드 프로토타입의 상세 설계입니다.

### 📊 최종 대시보드 UI/UX 프로토타입: 안전성 레벨 시각화

**1. 디자인 시스템 재확인**

*   **Primary Color (신뢰):** Deep Navy Blue (`#0A1931`)
*   **Accent Color (안전/성장):** System Green (`#10B981`)
*   **Text Color:** White (`#FFFFFF`) 또는 Light Gray (`#E5E7EB`)
*   **Data Visualization Style:** 데이터 기반의 신뢰감을 주기 위해, 단순한 막대 그래프 대신 **게이지(Gauge)** 및 **상태 표시등(Indicator Light)**를 중심으로 설계합니다.

**2. 핵심 시각화 요소: 안전성 레벨 (Safety Level)**

PTI와 헤징 ROI를 결합하여 사용자가 직관적으로 시스템의 상태를 파악할 수 있도록 '안전성 레벨'을 정의합니다.

| 티어 | 명칭 | PTI 기준 (시스템 안정성) | 헤징 ROI 기준 (수익화) | 시각적 색상 | 설명 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Basic** | Basic Shield | 70% 미만 (위험) | 낮은 수준의 ROI | **Red/Amber 경고** | 시스템 안정성이 낮아 리스크가 높음. 즉각적인 모니터링 필요. |
| **Pro** | Pro Hedger | 85% 이상 (안정) | 목표 ROI 달성 중 | **System Green** | 시스템이 안전하며, 수익화 목표를 효과적으로 관리하고 있음. |

**3. 대시보드 레이아웃 및 컴포넌트 상세 설계**

**A. 메인 영역: KPI 위젯 섹션 (상단)**

*   **레이아웃:** 2x2 그리드 또는 카드형 레이아웃으로 구성하여 정보의 계층을 명확히 합니다.
*   **컴포넌트 1: 시스템 안전성 게이지 (PTI 중심)**
    *   **제목:** 시스템 안정성 지수 (PTI)
    *   **시각화:** 원형 게이지(Gauge) 차트를 사용합니다.
        *   **중앙 값:** 현재 PTI (%) 표시.
        *   **색상 변화:** PTI 값이 70% 미만일 경우 경고 색상(Amber/Red)으로, 85% 이상일 경우 System Green으로 즉시 변경됩니다. (가장 강조)
    *   **보조 정보:** "안전성 레벨: [현재 상태]" 텍스트를 게이지 아래에 명확히 표시합니다.

*   **컴포넌트 2: 수익화 성과 위젯 (ROI 중심)**
    *   **제목:** 헤징 ROI (수익화 지표)
    *   **시각화:** 진행률 바(Progress Bar)와 목표 대비 차이 표시를 결합합니다.
        *   **진행률 바:** 현재 ROI가 목표 수익 달성률에 도달했는지 시각적으로 표현합니다.
        *   **상태 표시:** "목표 달성" 또는 "추가 관리 필요" 텍스트를 색상과 함께 표시합니다.

**B. 보조 영역: 상세 데이터 및 리스크 모듈 (하단)**

*   **레이아웃:** 좌측에 상세 데이터 테이블, 우측에 실시간 알림/리스크 요약 패널로 분리합니다.
*   **좌측 패널 (데이터 테이블):**
    *   PTI 및 ROI의 시간 흐름에 따른 변화 추이를 **라인 차트(Line Chart)**로 표시하여 과거 데이터와의 비교를 용이하게 합니다.
    *   각 데이터 포인트는 Deep Navy Blue 배경에 System Green으로 강조합니다.

*   **우측 패널 (리스크 & 액션):**
    *   **제목:** 실시간 리스크 알림
    *   **내용:** 시스템 안정성(PTI)이 임계치 이하로 떨어지거나, 헤징 ROI가 위험 범위에 진입했을 때 **빨간색 배너**를 띄우고 구체적인 조치 사항을 제시합니다. (예: "경고: PTI 65% 도달. 즉시 모니터링 필요.")

**C. 최종 디자인 요약 (Visual Flow)**

1.  **상단 헤더:** 브랜드 로고 및 사용자 정보
2.  **중앙 섹션 (가장 중요):** **PTI 게이지**와 **ROI 진행률 바**를 중앙에 배치하여 시스템의 현재 상태(안전성 + 수익성)를 3초 내에 파악하게 합니다. (Deep Navy Blue 배경 위에서 System Green이 강력한 시각적 신뢰를 제공)
3.  **하단 섹션:** 시간 흐름 데이터 및 실시간 리스크 알림을 상세히 제공합니다.
