## 웹앱 기술 스택 및 데이터 파이프라인 초기 타당성 분석 보고서

CEO의 지시사항과 Researcher의 분석 결과를 바탕으로, 수익형 웹앱 개발을 위한 최소 기능 제품(MVP)에 적합한 기술 스택과 데이터 파이프라인 구성을 제안합니다. 이 설계는 장기 목표인 '데이터 파이프라인/API 연결 안정화'를 염두에 두고 설계되었습니다.

### 1. 추천 기술 스택 (Recommended Tech Stack)

수익형 웹앱의 신뢰성, 확장성, 그리고 데이터 시각화 요구사항을 충족시키기 위해 다음과 같은 스택을 제안합니다.

| 영역 | 추천 기술 | 선택 이유 및 역할 |
| :--- | :--- | :--- |
| **프론트엔드 (Visualization)** | React.js (Next.js 프레임워크 사용 권장) + D3.js / TradingView 위젯 | 복잡한 금융 데이터(캔들스틱, 지표)의 실시간 시각화 및 사용자 친화적인 인터페이스 제공에 최적화됨. Next.js는 SEO와 빠른 배포를 지원하여 초기 시장 진입에 유리함. |
| **백엔드 (API & Logic)** | Python (FastAPI 프레임워크) | 데이터 처리(파이프라인), API 엔드포인트 구축, 그리고 금융 데이터 분석 라이브러리(Pandas, NumPy 등)와의 통합이 가장 용이하며 개발 속도가 빠름. |
| **데이터베이스 (Storage)** | PostgreSQL 또는 TimescaleDB | 시계열 데이터(Time-Series Data)의 안정적인 저장 및 복잡한 지표 계산을 위한 강력한 관계형/시계열 DB 지원. |
| **데이터 파이프라인** | Python 스크립트 + Cron/Airflow (초기) | 외부 API에서 데이터를 수집하고, 정제하며, 데이터베이스에 삽입하는 자동화된 워크플로우 구축의 기반. |

### 2. 데이터 파이프라인 설계 (Data Pipeline Design)

안정적인 수익형 웹앱 운영을 위해서는 **외부 API 의존도를 최소화**하고 **데이터 저장의 안정성**을 확보하는 것이 핵심입니다.

#### A. 데이터 수집 계층 (Ingestion Layer)

*   **소스:** 금융 데이터 제공 API (예: Polygon, Alpha Vantage 등 유료/무료 API 활용).
*   **프로토콜:** 백엔드(Python)에서 주기적으로 외부 API를 호출하여 데이터를 가져옵니다. **(API 키는 환경변수로 관리)**
*   **자동화:** Python 스크립트를 사용하여 특정 시간 간격(예: 1분, 일별)으로 데이터를 수집하는 스케줄링을 설정합니다.
