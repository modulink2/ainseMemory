## 🔍 알고리즘-UI 흐름 일치성 최종 검토 (Conceptual Alignment Review)

알고리즘의 목표(상승/우상향/급등 예측)가 UI에서 시각화되어야 할 데이터 흐름과 완벽하게 매칭되는지 확인했습니다.

**1. 핵심 지표 출력 매핑:**
*   **요구사항:** 알고리즘은 `상승`, `우상향`, `급등`의 세 가지 예측 결과를 실시간으로 계산하여 제공해야 합니다.
*   **Schema 일치 여부 검토:** JSON Schema는 이 세 가지 예측 결과와 각 예측에 대한 **신뢰도 점수(Confidence Score)** 및 **예측 시점(Timestamp)** 필드를 포함하도록 설계되었습니다. 코드는 모든 지표를 계산한 후, 이 구조화된 JSON 객체를 출력하도록 구현되어야 합니다.
*   **일치성 확인:** 코드의 최종 출력 포맷이 Designer가 요구한 `[Prediction_Type: Value, Confidence: Score, Timestamp: T]` 형태를 정확히 따르는지 검토했습니다. (실제 코드가 제공되지 않았으므로, 이 구조 준수를 최우선으로 설정합니다.)

**2. 데이터 흐름 일치성:**
*   **요구사항:** 실시간 스트리밍 데이터를 기반으로 예측값이 계산되어야 합니다.
*   **흐름 검토:** API 연동 단계에서 데이터 수신 지연(Latency) 및 데이터 누락 발생 시, 알고리즘은 **'데이터 불완전 상태'**를 명시하는 플래그(`Data_Integrity: OK/Warning/Error`)를 출력해야 합니다. 이는 UI가 사용자에게 잘못된 신호를 제공하는 것을 방지합니다.

## 🛡️ API 연동 에러 핸들링 계획 (API Integration Error Handling Plan)

키움 API(또는 기타 데이터 소스) 연동 시 발생할 수 있는 모든 예외 상황에 대비하여, 시스템이 비정상 상태에 빠지지 않고 사용자에게 명확한 피드백을 제공하도록 다음과 같이 에러 핸들링 계획을 문서화합니다.

### 1. 데이터 접근 및 인증 오류 (Authentication & Access Errors)

| 오류 유형 | 발생 원인 | 대응 전략 (Action) | UI/사용자 피드백 |
| :--- | :--- | :--- | :--- |
| **API Key 만료** | 설정된 API 키의 유효 기간이 만료됨. | 즉시 시스템을 중지하고 관리자에게 알림 전송. 재인증 프로세스 강제 시작. | "시스템 인증 오류. 관리자 확인 필요." |
| **권한 부족 (Permission Denied)** | 데이터 접근 권한(API Key)이 특정 종목 또는 시점에 없어 요청 거부됨. | 해당 종목/데이터에 대한 접근 권한을 재확인하고, 차단된 항목을 로그에 기록. | "요청하신 데이터에 접근할 권한이 없습니다." |
| **Rate Limit 초과** | 설정된 시간 내에 API 요청 횟수를 초과함. | 요청을 일시 중지(Backoff)하고, 재시도 로직을 적용하여 점진적으로 재시도. | "일시적으로 데이터 요청이 지연되고 있습니다. 잠시 후 다시 시도해주세요." |

### 2. 데이터 스트리밍 및 처리 오류 (Streaming & Processing Errors)

| 오류 유형 | 발생 원인 | 대응 전략 (Action) | UI/사용자 피드백 |
| :--- | :--- | :--- | :--- |
| **스트림 끊김 (Connection Loss)** | 서버 연결이 일시적으로 끊어짐. | 재연결 로직을 즉시 실행하여 데이터 스트림 복구 시도. 실패 시, 마지막 성공 지점부터 재요청. | "데이터 스트리밍에 일시적인 오류가 발생했습니다. 연결을 복구 중입니다." |
| **데이터 형식 불일치** | API에서 반환된 데이터(예: 가격, 거래량)의 형식이 예상 JSON Schema와 다름 (Null 또는 잘못된 문자열). | 데이터 수신 즉시 타입 검증(Type Casting)을 수행하고, 불일치 시 해당 데이터를 **'오류 플래그'**로 분류하여 예측 계산에서 제외. | "데이터 형식 오류가 감지되어 현재 예측값은 보류됩니다." |
| **알고리즘 입력 오류** | 예측에 필요한 필수 변수(과거 N일 데이터) 중 일부가 누락되었을 때. | 알고리즘 실행 전에 필수 데이터 존재 여부를 체크하고, 누락 시 '데이터 부족' 상태로 플래그 지정 후 계산 보류. | "예측을 위해 충분한 과거 데이터를 확보해야 합니다." |

### 3. 시스템 및 예외 오류 (System & Exception Errors)

| 오류 유형 | 발생 원인 | 대응 전략 (Action) | UI/사용자 피드백 |
| :--- | :--- | :--- | :--- |
| **알고리즘 계산 실패** | 수학적 연산 중 0으로 나누기 등 내부 로직 오류 발생. | 예외(Exception)을 포착하여 해당 종목의 예측 결과는 **'계산 불가'**로 표시하고, 로그에 상세한 스택 트레이스를 기록. | "알고리즘 계산 중 내부 오류가 발생했습니다." |
| **데이터베이스 연결 실패** | 실시간 데이터 저장 또는 로드 과정에서 DB 연결이 끊어짐. | 트랜잭션을 롤백하고, 재시도 메커니즘을 통해 연결을 복구하거나 해당 요청을 실패로 처리. | "데이터 저장 중 시스템 오류가 발생했습니다." |
