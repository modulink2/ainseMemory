### 💡 Error State 컴포넌트 3안 비교 및 최종 제안

Writer가 정의할 것으로 예상되는 주요 에러 유형(데이터 로딩 실패, 예측 알고리즘 오류, API 연결 실패)에 맞춰 세 가지 시나리오를 디자인했습니다.

#### **[안 1] 데이터 로딩/API 연결 실패 (Critical Failure)**
*   **상황:** 실시간 데이터 스트리밍 또는 초기 데이터 로딩에 실패했을 때 (가장 심각한 오류).
*   **목표:** 즉각적인 조치를 요구하고, 시스템이 작동하지 않음을 명확히 알립니다.
*   **디자인 제안:** **Red Alert State (경고/위험)**

| 속성 | 상세 내용 | 코드 및 레이아웃 |
| :--- | :--- | :--- |
| **상태 색상** | `#FF4136` (진한 레드) | 배경: `bg-red-50`, 텍스트: `text-red-800` |
| **아이콘** | ⚠️ 느낌표 아이콘 (`fa-exclamation-triangle`) | 중앙 배치, 크기 크게 |
| **제목** | **🚨 데이터 연결 오류** | 볼드체, 시스템 메시지 명확화 |
| **본문** | "API 서버 연결에 실패했습니다. 잠시 후 다시 시도해 주세요." | 일반 텍스트, 해결책 제시 |
| **액션 버튼** | [재연결 시도] (Primary Action) | `bg-red-600`, `hover:bg-red-700` |

#### **[안 2] 예측 알고리즘 오류 (Algorithm Failure)**
*   **상황:** 데이터는 성공적으로 로드되었으나, 예측 모델(상승/우상향/급등) 계산 과정에서 내부적인 논리적 모순이 발생했을 때.
*   **목표:** 시스템은 작동 중이나 결과의 신뢰성에 문제가 있음을 알리고, 사용자에게 재검토를 요청합니다.
*   **디자인 제안:** **Amber Warning State (주의/재검토 필요)**

| 속성 | 상세 내용 | 코드 및 레이아웃 |
| :--- | :--- | :--- |
| **상태 색상** | `#FFC107` (앰버/노란색) | 배경: `bg-amber-50`, 텍스트: `text-amber-800` |
| **아이콘** | 🧐 느낌표 아이콘 (`fa-search`) 또는 ⚖️ 저울 아이콘 | 중앙 배치, 분석 중임을 암시 |
| **제목** | **⚠️ 예측 결과 검토 필요** | 볼드체, 분석 상태 명확화 |
| **본문** | "알고리즘 계산 과정에서 데이터 불일치가 감지되었습니다. 결과를 재검토해 주세요." | 시스템이 오류를 감지했음을 알림 |
| **액션 버튼** | [상세 로그 보기] (Secondary Action) | `bg-amber-600`, 텍스트: `text-amber-800` |

#### **[안 3] 데이터 불완전/경고 상태 (Partial Data State)**
*   **상황:** 일부 지표(예: 우상향 예측은 가능하나 급등 예측 데이터가 누락)는 성공적으로 로드되었으나, 전체 정보가 완성되지 않았을 때.
*   **목표:** 부분적인 정보만 제공되므로, 사용자에게 어떤 부분이 누락되었는지 구체적으로 알려줍니다.
*   **디자인 제안:** **Blue Info State (정보 제공)**

| 속성 | 상세 내용 | 코드 및 레이아웃 |
| :--- | :--- | :--- |
| **상태 색상** | `#1762A3` (갓더주식 블루) | 배경: `bg-blue-50`, 텍스트: `text-blue-800` |
| **아이콘** | ℹ️ 정보 아이콘 (`fa-info-circle`) | 중앙 배치, 정보 제공 역할 강조 |
| **제목** | **ℹ️ 일부 데이터만 표시됨** | 명확한 상태 구분 |
| **본문** | "급등 예측 지표는 현재 실시간 데이터 부족으로 표시할 수 없습니다." | 누락된 항목을 구체적으로 설명 |
| **액션 버튼** | [새로고침] (Soft Action) | `bg-blue-600`, `hover:bg-blue-700` |

### ✨ 최종 디자인 시스템 요약 및 적용 가이드

이 세 가지 상태는 갓더주식의 신뢰감을 유지하면서도, 오류 상황에 따라 사용자에게 가장 적절한 행동을 유도하도록 설계되었습니다.

| 상태 이름 | 색상 코드 (Primary) | 목적 | 핵심 액션 |
| :--- | :--- | :--- | :--- |
| **Critical Error** | `#FF4136` (Red) | 시스템 작동 불가 | [재연결 시도] |
| **Warning State** | `#FFC107` (Amber) | 결과의 신뢰성 문제 | [상세 로그 보기] |
| **Info State** | `#1762A3` (Blue) | 데이터의 불완전성 | [새로고침] |

이 디자인은 Writer가 구현할 JSON Schema 필드에 따라 동적으로 적용되어야 하며, 특히 **Warning State**에서는 해당 오류가 어떤 예측 지표와 연결되는지 명시하는 것이 핵심입니다.
