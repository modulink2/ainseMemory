# 🎨 최종 Error State 디자인 확정: API 통합 에러 핸들링

## 1. 디자인 원칙 (Brand Alignment)
*   **톤앤매너:** 신뢰감 있고 차분함. 혼란을 최소화하고 명확한 해결책을 제시한다.
*   **컬러 사용:** 주황색 계열(경고), 빨간색 계열(치명적 오류), 파란색 계열(정보/재시도)을 사용하여 심각도에 따라 시각적으로 구분한다.
*   **타이포그래피:** 명확하고 읽기 쉬운 산세리프체 사용.

## 2. 에러 상태별 상세 디자인 스펙

### A. [Critical Error] - 인증 실패 (HTTP 401 Unauthorized)
**상황:** API 키 또는 인증 토큰이 유효하지 않을 때.
**UX 목표:** 사용자가 즉시 재인증 절차를 시작하도록 유도.

*   **레이아웃:** 전체 화면을 덮는 모달(Modal) 형태로 표시. 배경은 어둡게 처리하여 오류에 집중하게 함.
*   **아이콘/색상:** **빨간색** 경고 아이콘 (❌ 또는 ⚠️).
*   **제목:** "인증 오류 발생" (굵고 명확하게)
*   **메시지:** "API 키 또는 인증 토큰이 유효하지 않습니다."
*   **Actionable Feedback (핵심):**
    1.  **주요 버튼:** **[재로그인 및 인증]** (Primary Action, 파란색 또는 강조 색상)
    2.  **보조 옵션:** [API 키 관리 페이지로 이동] (Secondary Action, 텍스트 링크)
*   **흐름:** 사용자가 [재로그인 및 인증]을 클릭하면, 로그인 화면으로 즉시 리디렉션되도록 연결한다.

### B. [Server Error] - 서버 내부 오류 (HTTP 500 Internal Server Error)
**상황:** 데이터 처리 중 시스템에서 예상치 못한 오류가 발생했을 때.
**UX 목표:** 사용자에게 양해를 구하고 재시도 기회를 제공하며, 백엔드 팀의 개입 필요성을 암시.

*   **레이아웃:** 화면 중앙에 배치되는 카드 형태의 알림 창 (Notification Card).
*   **아이콘/색상:** **주황색** 경고 아이콘 (⚠️).
*   **제목:** "시스템 오류 발생"
*   **메시지:** "데이터 처리 중 서버 오류가 발생했습니다. 잠시 후 다시 시도해 주세요."
*   **Actionable Feedback (핵심):**
    1.  **주요 버튼:** **[다시 시도하기]** (Primary Action, 파란색) - 3초 지연 후 재시도 로직 실행.
    2.  **보조 옵션:** [관리자에게 오류 보고하기] (Secondary Action, 텍스트 링크) - 상세 에러 코드를 포함하여 서버 로그 시스템으로 연결.
*   **흐름:** 사용자가 [다시 시도하기]를 클릭하면, 로딩 스피너가 표시되며 백그라운드에서 API 호출을 재실행한다.

### C. [Timeout Error] - 요청 시간 초과 (HTTP 408 Request Timeout)
**상황:** 네트워크 지연 또는 서버 응답이 너무 오래 걸렸을 때.
**UX 목표:** 사용자가 네트워크 상태를 확인하고 재연결을 시도하도록 안내.

*   **레이아웃:** 화면 상단에 위치하는 작은 배너 또는 인라인 메시지 형태.
*   **아이콘/색상:** **파란색** 정보 아이콘 (ℹ️).
*   **메시지:** "데이터 요청 시간 초과. 네트워크 연결을 확인해주세요."
*   **Actionable Feedback (핵심):**
    1.  **주요 버튼:** **[네트워크 상태 확인]** (Primary Action, 파란색) - 브라우저/시스템의 인터넷 연결 상태를 팝업으로 보여줄 수 있음.
    2.  **보조 옵션:** [새로고침] (Simple Action)
*   **흐름:** 사용자가 [네트워크 상태 확인]을 클릭하면, 네트워크 상태 진단 인터페이스가 표시되며, 성공 시 자동 재연결을 시도한다.

### D. [Input Validation Error] - 입력값 형식 오류 (HTTP 400 Bad Request)
**상황:** 수량, 가격 등 사용자가 입력한 파라미터가 시스템 규칙을 위반했을 때.
**UX 목표:** 오류가 발생한 **구체적인 필드**를 명시하고 즉각적인 수정이 가능하도록 안내.

*   **레이아웃:** 입력 폼(Form) 내에서 해당 오류가 발생한 필드와 메시지를 직접 표시 (Inline Validation).
*   **아이콘/색상:** **빨간색** 테두리 및 텍스트로 오류 필드를 즉시 강조.
*   **메시지:** "주문 요청 형식 오류. 수량 또는 가격 형식이 잘못되었습니다."
*   **Actionable Feedback (핵심):**
    1.  **구체적 피드백:** **"가격(Price) 필드에 잘못된 형식입니다. 숫자만 입력해 주세요."** (오류가 발생한 필드 바로 아래에 표시).
    2.  **수정 방법:** 오류가 발생한 필드 주변에 빨간색 테두리를 적용하고, 툴팁으로 수정 가이드를 제공한다.
*   **흐름:** 사용자가 해당 필드를 수정하면, 시스템은 실시간으로 유효성을 검사하여 입력 즉시 오류를 해소하거나 새로운 오류를 표시한다.

## 3. 비주얼 시스템 반영 (가이드라인)

| 요소 | 정의 | 적용 방식 |
| :--- | :--- | :--- |
| **Error State Card** | 서버 에러 (5xx)에 사용됨. 명확한 행동 유도. | 배경: Light Gray (#F0F0F0). 카드 테두리: 얇은 주황색. |
| **Critical Alert** | 인증 오류 (401). 즉각적인 재인증 요구. | 배경: Dark Gray (#333333). 아이콘: Bright Red. CTA 버튼은 Accent Color(Blue)로 강조. |
| **Validation Field Error** | 입력값 오류 (400). 사용자 입력 위치에 직접 표시. | 텍스트 색상: Red (#D93025). 필드 주변에 강한 경고 테두리. |
