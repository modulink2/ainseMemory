## 🎨 최종 에러 상태 디자인 사양 (Error State Design Specification)

### 1. 브랜드 컬러 시스템 매핑 (Color Mapping)

오류의 심각도에 따라 브랜드의 핵심 색상을 사용하여 즉각적인 시각적 인식을 제공합니다.

| 오류 심각도 | 상태 명칭 | 배경색 (Background) | 텍스트 색상 (Text Color) | 강조선/아이콘 색상 (Accent Color) | 적용 기준 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Critical** | 치명적 오류 (Fatal Error) | `#FEE2E2` (밝은 적색 계열) | `#D32F2F` (진한 빨강) | `#D32F2F` | 자동 매매 실패, API 연결 두절 등 즉시 조치가 필요한 상황 |
| **Warning** | 경고 오류 (Warning Error) | `#FFFDE7` (밝은 주황 계열) | `#F57C00` (주황색) | `#F57C00` | 데이터 지연, 조건 미충족 등 사용자 주의가 필요한 상황 |
| **Info** | 정보 메시지 (Info Message) | `#E8F5E9` (밝은 녹색 계열) | `#388E3C` (짙은 녹색) | `#388E3C` | 성공적인 작업 완료 또는 시스템 상태 안내 |

### 2. 컴포넌트 레이아웃 및 스타일 가이드 (Layout & Style Guide)

모든 에러 메시지 컴포넌트는 일관된 간격과 구조를 유지하여 개발팀의 구현 용이성을 높입니다.

**A. 공통 스타일 정의:**
*   **패딩 (Padding):** `16px` (수평), `12px` (수직)
*   **여백 (Margin):** `10px` (외부 여백), `8px` (내부 요소 간격)
*   **폰트 스타일:** 오류 텍스트는 항상 `Pretendard Bold, 16px`, 설명 텍스트는 `Pretendard Regular, 14px`를 사용합니다.

**B. Critical Error 컴포넌트 (`ErrorState_Critical`) 레이아웃:**
*   **구조:** 중앙 정렬 (Center Alignment)
*   **배경:** `#FEE2E2`
*   **아이콘:** 크기 `48px`, 색상 `#D32F2F` (에러 심볼: ❌ 또는 🚨 사용)
*   **메시지 영역:** 최대 5줄, 중앙 정렬. 가장 중요한 행동 유도 문구(Actionable Feedback)를 최상단에 배치합니다.
    *   *예시 레이아웃:* [아이콘] → **[헤드라인: Critical Error]** → [설명: API 연결 실패] → [액션 버튼: 재시도하기 (Primary)]

**C. Warning Error 컴포넌트 (`ErrorState_Warning`) 레이아웃:**
*   **구조:** 좌측 정렬 (Left Alignment)
*   **배경:** `#FFFDE7`
*   **아이콘:** 크기 `32px`, 색상 `#F57C00` (경고 심볼: ⚠️ 사용)
*   **메시지 영역:** 왼쪽 정렬. 경고의 원인을 명확히 설명하고 해결책을 제시합니다.
    *   *예시 레이아웃:* [아이콘] → **[헤드라인: Warning - 데이터 지연]** → [설명: 실시간 데이터 업데이트가 5초 지연되었습니다.] → [액션 버튼: 확인 및 대기 (Secondary)]

**D. Info Message 컴포넌트 (`MessageState_Info`) 레이아웃:**
*   **구조:** 일반 텍스트 블록
*   **배경:** `#E8F5E9`
*   **스타일:** 헤드라인 없이 순수 정보 전달에 집중합니다. 시스템 상태를 부드럽게 알립니다.

### 3. 최종 UI 개발팀 전달 사양 요약 (Summary for Dev Team)

개발팀은 다음의 구체적인 값을 기반으로 컴포넌트를 구현해야 합니다.

1.  **컬러 팔레트:** 위 표에 정의된 Hex 코드를 시스템 전체에 적용합니다.
2.  **레이아웃 규칙:** Critical은 중앙 집중식 행동 유도, Warning은 상세 설명 중심의 좌측 정렬을 기본 원칙으로 삼습니다.
3.  **Actionable Feedback 우선순위:** 모든 에러 컴포넌트에서 가장 중요한 것은 **"사용자가 다음으로 취해야 할 행동(Action)"**을 명확히 제시하는 것입니다. (예: '재시도', '설정 변경', '로그아웃' 등)
4.  **간격 준수:** 내부 요소 간의 모든 패딩과 마진은 정의된 `16px`, `12px` 규칙을 엄격하게 따릅니다.
