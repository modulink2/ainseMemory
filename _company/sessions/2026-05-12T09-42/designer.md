## 🖼️ Error State 최종 적용 및 시각적 검증 보고서

**목표:** 자동 매매 시스템의 모든 예외 상황에서 사용자에게 명확하고 행동 유도적인 피드백을 제공하는 UI 컴포넌트 완성.

### 1. 핵심 디자인 원칙 재확인 (브랜드 일관성)
*   **톤앤매너:** 주식 투자에 대한 신뢰감과 긴박감을 동시에 전달하며, **갓더주식**의 전문성과 명확성을 유지해야 함.
*   **색상 활용:** 오류의 심각도에 따라 색상을 계층화하여 사용자가 즉각적으로 위험 수준을 인지하도록 한다.

### 2. Error State 세부 사양 (실제 적용 기준)

| 에러 상태 | 심각도 | 배경색 (Background) | 텍스트 색상 (Text Color) | 아이콘/표시 | 행동 유도 메시지 (Actionable Feedback) | 적용 범위 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **Critical Error** | 최상 (즉각 조치 필요) | `#d9534f` (빨강 계열) | `#ffffff` (흰색) | ⚠️ (경고 아이콘) | "시스템 오류 발생. 즉시 관리자에게 알림을 보내십시오." | 전체 화면 오버레이 |
| **Warning** | 중상 (주의 필요) | `#f0ad4e` (주황 계열) | `#333333` (짙은 회색) | 🔔 (알림 아이콘) | "데이터 스트리밍 지연 발생. 실시간 데이터 수신을 확인하십시오." | 특정 모듈/차트 영역 |
| **Error** | 중 (오류 발생) | `#5bc0de` (파랑 계열) | `#ffffff` (흰색) | ❌ (오류 아이콘) | "매매 실행 실패. API 연결 상태를 점검하십시오." | 매매 실행 버튼 및 상세 로그 영역 |
| **Info/Success** | 하 (정보 제공) | `#dff0d8` (연두 계열) | `#3c763d` (녹색) | ✅ (성공 아이콘) | "모든 데이터가 성공적으로 수신되었습니다." | 상태 표시줄 및 성공 알림창 |

### 3. 사용자 흐름 시각적 검증 (UX Flow Validation)

**A. 예측 결과 화면에서의 적용:**
*   **상황:** 알고리즘이 예측을 수행하는 중 발생한 지연 또는 실패 상황.
*   **검증 사항:** `Warning` 상태가 활성화될 때, 사용자가 매매를 진행하지 않고 **데이터 확인**에 집중하도록 유도하는 UI 레이아웃(예: 중앙 정렬된 경고 박스)이 최적화되었는지 확인했습니다.

**B. 자동 매매 실행 시의 적용:**
*   **상황:** `Error` 상태가 발생하여 자동 매매 명령이 실패한 경우.
*   **검증 사항:** 사용자가 즉시 취할 수 있는 행동(재시도 버튼, 로그 보기 링크)이 **Critical Error** 상태보다 명확하게 강조되었는지 확인했습니다. (Actionable Feedback의 우선순위 검증 완료.)

**C. 최종 사용자 경험 점검:**
*   **결론:** 확정된 색상과 메시지는 주식 투자 환경에서 요구되는 긴박감(빨강/주황)과 정보 전달력(파랑/녹색)을 적절히 혼합하여, 사용자가 **혼란 없이 다음 행동을 결정**할 수 있도록 명확하게 설계되었습니다.
