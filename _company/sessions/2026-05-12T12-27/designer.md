## 2. Error State 유형별 상세 명세

Writer의 보고서에 따라, 시스템 오류 발생 시 사용자에게 전달할 정보와 디자인은 다음 세 가지 레벨로 분류하여 설계합니다.

### A. Critical Error (치명적 오류: 자동 매매 실패, API 통신 완전 실패)

*   **발생 조건:** 키움 API 연동 실패, 데이터 스트리밍 중 치명적 오류 발생 등 시스템 기능이 마비되는 상황.
*   **표시 형태:** 화면 전체 또는 핵심 영역에 강력하게 표시.
*   **색상:** `Error Color` (`#dc3545`) 배경, 흰색 텍스트.
*   **메시지 (예시):** "🔴 **시스템 오류 발생**: 실시간 데이터 수신에 실패했습니다. 서버 연결을 확인해주세요."
*   **레이아웃:** 중앙 정렬, 최대 폭 전체를 차지하며 시각적 강조.

### B. Warning Error (경고 오류: 예측값의 신뢰도 하락 또는 잠재적 위험)

*   **발생 조건:** 예측 알고리즘이 불안정한 입력 데이터를 받거나, 특정 종목에 대한 데이터 수신 지연 발생 등, 즉시 행동은 필요하나 시스템 전체가 마비되지는 않은 상황.
*   **표시 형태:** 해당 데이터 섹션에만 경고 표시.
*   **색상:** `Warning Color` (`#ffc107`) 배경 또는 테두리, 검은색 텍스트.
*   **메시지 (예시):** "⚠️ **데이터 주의**: 실시간 예측값의 신뢰도가 낮습니다. 데이터 수신을 재시도해주세요."
*   **레이아웃:** 해당 종목/차트 영역 주변에 노란색 테두리 및 아이콘 표시.

### C. Informational Error (정보 오류: 일반적인 입력 오류 또는 처리 실패)

*   **발생 조건:** 사용자가 잘못된 형식으로 데이터를 입력했거나, 특정 설정 변경이 시스템에 반영되지 않은 경우 등 비치명적인 사용자 입력 오류.
*   **표시 형태:** 미세한 텍스트 알림 형태로 표시.
*   **색상:** `Primary Color` (`#007bff`) 또는 회색 계열.
*   **메시지 (예시):** "ℹ️ **입력 오류**: 입력하신 종목 코드가 유효하지 않습니다."
*   **레이아웃:** 입력 필드 바로 아래에 작은 텍스트로 표시하며, 마우스 오버 시 상세 설명 툴팁 제공.

## 3. 최종 개발팀 전달 사양 요약 (Summary for Dev Team)

개발팀은 위 명세에 따라 다음과 같은 컴포넌트별 스타일링을 구현해야 합니다.

1.  **Global Error Handler:** 모든 에러 메시지는 **'Critical/Warning/Informational'** 상태 코드를 기반으로 `Error Color`, `Warning Color`, `Primary Color`를 적용하여 렌더링한다.
2.  **Data-Specific Feedback:** 실시간 예측 데이터 영역에는 A(Critical) 또는 B(Warning) 상태가 발생했을 경우, 해당 데이터를 감싸는 컨테이너에 명시된 색상과 메시지를 오버레이 한다.
3.  **Accessibility (A11y):** 모든 오류 메시지에는 스크린 리더를 고려하여 명확한 `aria-live` 영역을 설정하고, 텍스트뿐만 아니라 시각적 강조(색상)로도 상태를 인지할 수 있도록 한다.
