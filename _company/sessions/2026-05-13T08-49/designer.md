## 📊 대시보드 최종 디자인 사양 및 개발 우선순위 확정

이 문서는 **시스템 신뢰도(KPI) 시각화**를 최우선 목표로 하며, 파이썬 애플리케이션에 직접 반영될 수 있도록 구체적인 컴포넌트 사양을 정의합니다.

### 1. 핵심 자산 (KPI 기반 디자인 원칙)

| KPI 지표 | 측정 기준 | 시각적 강조 방식 | 색상 코드 (Primary/Warning) | 중요도 |
| :--- | :--- | :--- | :--- | :--- |
| **오류율 (Error Rate)** | API 실패 및 로직 오류 발생 빈도 | 대시보드 상단에 Critical Alert로 표시 | `#D9534F` (Critical Red) | 최우선 |
| **지연 시간 (Latency)** | 데이터 수신 및 처리 지연 시간 | 실시간 그래프(Gauge/Line Chart)로 시각화 | `#F0AD4E` (Warning Orange) | 높음 |
| **시스템 안정성 점수** | 오류율과 지연 시간을 종합한 통합 지표 | 중앙 대형 KPI 카드에 표시 | `#5CB85C` (Success Green) | 핵심 |

### 2. UI/UX 프로토타입 최종 사양 (DashboardLayout 기준)

**레이아웃:** `DashboardLayout` (사이드바 + 탑바 + 메인 영역 통합) 준수.

#### A. 탑바 (Header Bar)
*   **구성:** 시스템 상태 표시 및 빠른 접근 메뉴.
*   **요소:** 현재 시스템 안정성 점수(System Stability Score), 마지막 데이터 업데이트 시간, 알림 아이콘.
*   **디자인 톤:** 신뢰감을 주는 다크 모드(Dark Mode) 기반으로 설정하며, KPI 색상(Red/Orange/Green)을 상태 표시등처럼 활용합니다.

#### B. 메인 영역 (Main Content Area) - 핵심 시각화 구역
1.  **KPI 대시보드 카드 (Focus Area):**
    *   **오류율:** 현재 오류율 % 및 지난 24시간 평균 대비 변화율을 표시. (Critical Red 강조)
    *   **지연 시간:** 평균 지연 시간 (ms) 및 최대 지연 시간(ms)을 표시. (Warning Orange 강조)
    *   **안정성 점수:** 종합 안정성 점수 (0~100점)를 중앙에 배치하여 시스템의 전반적인 건강 상태를 한눈에 파악하도록 합니다.

2.  **실시간 트렌드 그래프 (Real-time Trend Chart):**
    *   **목표:** 오류율과 지연 시간의 추이 변화를 시간 경과에 따라 시각화합니다.
    *   **구현 방식:** 두 개의 독립된 라인 차트(오류율 추이, 지연 시간 추이)를 플로팅하여, 상호 연관성을 파악할 수 있도록 배치합니다.

3.  **알림 및 액션 영역 (Actionable Feedback):**
    *   KPI가 임계값(Threshold)을 벗어날 경우, 해당 카드에 **Actionable Feedback** 메시지(예: "오류율 1% 초과 감지, 즉시 점검 필요")를 팝업 또는 인라인으로 제공합니다. (Designer 메모리 기반)

### 3. 개발 우선순위 (Implementation Roadmap)

시스템 안정성 KPI를 기반으로 **MVP(Minimum Viable Product)** 구현을 위한 단계를 다음과 같이 확정하고 즉시 착수합니다.

| 순서 | 작업 내용 | 목표 산출물 | 담당 에이전트 | 예상 소요 시간 |
| :--- | :--- | :--- | :--- | :--- |
| **P1 (Foundation)** | **핵심 데이터 구조 및 API 연동 모듈 정의** | KPI 측정 로직(오류율, 지연 시간)을 실시간으로 기록하고 반환하는 백엔드 모듈 설계 완료. | 코다리 | 2일 |
| **P2 (Visualization)** | **KPI 대시보드 컴포넌트 구현** | 확정된 색상 코드와 레이아웃 좌표를 반영한 KPI 카드(오류율, 지연 시간)의 UI 컴포넌트(CSS/React Component) 구현. | Designer | 3일 |
| **P3 (Integration)** | **실시간 데이터 스트림 연동 및 시각화** | P1에서 정의된 데이터 흐름을 기반으로 실시간 데이터를 받아와 P2에서 디자인한 컴포넌트에 정확히 매핑하여 시각화 구현. | 코다리 & Designer | 4일 |
| **P4 (Feedback)** | **Actionable Feedback 시스템 구축** | KPI 임계값 초과 시 사용자에게 명확하고 행동 유도적인 경고 메시지(Alert)가 표시되는 UI 흐름 완성. | Designer | 2일 |

### 🚀 즉시 구현 계획 착수

1.  **코다리에게 P1 작업 지시:** 실시간 KPI 측정 로직 및 데이터 기록 메커니즘의 초기 코드를 작성하도록 지시합니다.
2.  **Designer에게 P2/P3 디자인 사양 전달:** 확정된 KPI 시각화 사양(색상, 레이아웃)을 기반으로 개발팀이 즉시 적용할 수 있는 컴포넌트 구현 상세 명세(CSS/Layout Coordinates)를 최종 정리하여 전달하도록 지시합니다.
