## 📊 대시보드 안정성 시각화 디자인 사양 (KPI 강조 모듈)

### 1. 디자인 원칙 및 브랜드 연계
*   **톤앤매너:** 갓더주식의 정체성을 반영하여, 데이터는 명확하고 신뢰감을 주어야 하며, 불안정할 때는 경고를 통해 즉각적인 조치를 유도해야 합니다. (신뢰성 기반)
*   **컬러 시스템 적용:** 기존 브랜드 컬러(만약 정의되었다면)와 대비되는 색상 팔레트를 사용하여 KPI 상태를 명확히 구분합니다.
    *   **🟢 안정 (Stable):** 짙은 녹색 계열 (예: `#28a745` 또는 그보다 깊은 톤) - 시스템이 정상 범위 내에 있음을 나타냄.
    *   **🟡 주의 (Warning):** 주황/노란색 계열 (예: `#ffc107`) - 경고 상태, 임계치 근접을 나타냄.
    *   **🔴 위험 (Critical):** 빨간색 계열 (예: `#dc3545`) - 심각한 오류 또는 시스템 중단 가능성을 나타냄.

### 2. 핵심 KPI 시각화 컴포넌트 정의

| KPI 항목 | 측정 기준 | 시각화 방식 | 색상 코드 (상태) | 우선순위 | 설명 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **오류율 (Error Rate)** | $\le 0.5\%$ 목표 | 게이지/반응형 바 (Progress Bar) | 🟢 안정: 녹색, 🟡 주의: 노란색, 🔴 위험: 빨간색 | **P1** | 시스템의 전반적인 오류 발생 빈도를 직관적으로 표시. |
| **지연 시간 (Latency)** | $\pm 5\%$ 목표 | 시계/실시간 숫자 표시 (Gauge/Digital Readout) | 🟢 안정: 녹색, 🟡 주의: 노란색, 🔴 위험: 빨간색 | **P1** | 데이터 처리 속도를 실시간으로 보여주어 알고리즘 반응성을 강조. |
| **시스템 상태** | E2E 테스트 결과 | 명확한 상태 표시 (Badge/Status Indicator) | 🟢 정상(Green), 🟡 경고(Yellow), 🔴 실패(Red) | **P1** | 현재 시스템의 통합 상태를 한눈에 파악. |

### 3. 컴포넌트 개발 우선순위 확정

시스템 신뢰도 전달을 최우선 목표로 설정하고, 사용자에게 가장 즉각적인 행동 유도를 제공하는 컴포넌트부터 개발합니다.

1.  **P1: 시스템 상태 표시 컴포넌트 (Status Indicator Component)**
    *   **목표:** 현재 시스템의 통합 안정성(E2E Test 결과)을 가장 먼저 명확히 전달한다.
    *   **구현 상세:** 녹색/노란색/빨간색 배지(Badge)를 사용하여 전체 상태를 표시하고, 클릭 시 상세 로그나 경고 메시지를 팝업으로 연결하는 인터페이스를 설계한다. (이는 코다리가 구현한 KPI 데이터를 직접적으로 반영)

2.  **P2: 실시간 KPI 게이지 컴포넌트 (Real-time KPI Gauge Component)**
    *   **목표:** 오류율과 지연 시간을 실시간으로 시각화하여 알고리즘의 신뢰도를 지속적으로 입증한다.
    *   **구현 상세:** 정의된 색상 시스템을 적용한 게이지 차트를 구현하며, 목표치(0.5%, $\pm 5\%$) 대비 현재 상태를 명확히 보여주어야 한다.

3.  **P3: 추이 분석 그래프 (Trend Analysis Chart)**
    *   **목표:** 과거 데이터의 안정성 추이를 제공하여 장기적인 신뢰도를 구축한다.
    *   **구현 상세:** 시간 경과에 따른 오류율 및 지연 시간의 변화를 라인 차트(Line Chart)로 시각화하며, 이상 징후 발생 시 색상 변화를 통해 위험을 강조한다.
