## 📊 키움 RestApi 기반 주식 검색 알고리즘 시스템: 핵심 Feature Set 및 데이터 요구사항 정의

본 시스템은 **상승 예측, 급등 예측, 현재 강성주** 분야를 파악하는 것을 목표로 하므로, 실시간 가격 데이터와 기술적 지표(Technical Indicators)의 통합 분석이 필수적입니다.

### 1. 핵심 기능 (Feature Set) 목록

| ID | Feature 명칭 | 설명 | 주요 목표 |
| :---: | :--- | :--- | :--- |
| F1 | **실시간 데이터 수집 모듈** | 키움 RestApi를 통해 실시간 시세 및 기본 정보를 지속적으로 수집하고 정제하는 기능. | 데이터의 정확성과 지연 시간 최소화 |
| F2 | **기술적 분석 지표 계산 모듈** | 수집된 가격 데이터를 기반으로 이동평균선(MA), RSI, MACD 등 핵심 기술적 지표를 실시간으로 계산하는 기능. | 정량적 매매 신호 생성의 기초 마련 |
| F3 | **상승/하락 추세 분류 모듈** | 계산된 지표들을 조합하여 특정 종목이 '상승 추세', '하락 추세', '횡보 추세' 중 어디에 속하는지 분류하는 기능. | 시장 방향성 판단 |
| F4 | **급등/강성주 필터링 모듈** | 특정 조건(예: 급격한 거래량 변화, 특정 지표의 과매수 상태)을 만족하는 종목을 선별하여 '급등 후보' 또는 '강성주'로 분류하는 기능. | 투자 기회 식별 |
| F5 | **검색 및 알림 모듈** | 사용자가 정의한 복합적인 검색 조건(다중 지표 조합)에 따라 필터링하고, 조건 충족 시 사용자에게 즉시 알림을 제공하는 기능. | 사용자 맞춤형 정보 제공 |

### 2. 각 Feature별 데이터 요구사항 (Data Requirements)

각 기능이 정상적으로 작동하기 위해 시스템이 요구하는 데이터의 종류와 형태를 정의합니다.

#### ⚙️ F1: 실시간 데이터 수집 모듈 요구사항
*   **데이터 유형:** 현재가(시가, 고가, 저가, 종가), 거래량, 체결 시간.
*   **요구 사항:** 키움 RestApi에서 제공하는 **실시간 시세 스트림**에 대한 안정적인 연결 및 오류 처리 메커니즘.
*   **데이터 형식:** Timestamp (UTC/KST), Security Code, Price (Float), Volume (Integer).

#### ⚙️ F2: 기술적 분석 지표 계산 모듈 요구사항
*   **데이터 유형:** 시계열 데이터 (최소 N일 또는 N봉의 종가 및 거래량).
*   **요구 사항:** 과거 데이터를 저장할 수 있는 **시계열 데이터베이스(Time-Series DB)** 구조 설계.
*   **필수 지표 (예시):**
    *   **이동평균선 (MA):** 단기(5일, 20일), 중기(60일) 등 다양한 기간 설정 가능해야 함.
    *   **상대강도지수 (RSI):** 특정 기간(14일 기준) 계산 및 범위 정의 필요.
    *   **이동평균 수렴/확산 지표 (MACD):** MACD 선, 시그널 선, 히스토그램 값 저장 구조 필요.

#### ⚙️ F3: 상승/하락 추세 분류 모듈 요구사항
*   **데이터 유형:** F2에서 계산된 모든 기술적 지표의 **상대적인 변화율**.
*   **요구 사항:** 다중 지표 간의 **상관관계 분석 프레임워크**가 필요하며, 이를 통해 단순한 단일 지표 기반이 아닌 복합적인 추세 판단 로직을 구현해야 함.
*   **데이터 형식:** 추세 분류 결과 (Categorical: 상승/하락/횡보) 및 해당 분류의 신뢰도 점수.

#### ⚙️ F4: 급등/강성주 필터링 모듈 요구사항
*   **데이터 유형:** 실시간 데이터(F1)와 지표 변화율(F2, F3)의 **동시 비교**.
*   **요구 사항:** '급등'을 정의하기 위한 **거래량 대비 가격 변화율 임계값 (Threshold)** 및 **변동성 측정 지표 (Volatility Index)**의 데이터 요구.
*   **데이터 형식:** 필터링된 종목 목록, 각 종목별 급등/강성 점수.

#### ⚙️ F5: 검색 및 알림 모듈 요구사항
*   **데이터 유형:** 사용자가 설정한 **검색 조건(Parameters)**과 시스템이 계산한 **현재 상태(Results)**.
*   **요구 사항:** 동적이고 유연한 **쿼리 엔진** 구현. 사용자가 `[MA20 > MA60] AND [RSI < 70]`와 같은 논리식을 입력하면, 시스템이 이를 데이터베이스 검색어로 변환하여 결과를 반환해야 함.
*   **데이터 형식:** 사용자 정의 조건(JSON/YAML 형태), 필터링 결과 리스트.
