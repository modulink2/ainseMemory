### 🛠️ 구현 계획: 데이터 무결성 기반 리스크 헷지 자동 실행 통합

본 계획은 '데이터 무결성 검증 모듈'의 실패 신호를 받아, '리스크 헷지 모듈'이 즉각적으로 안전장치를 발동하도록 하는 **중앙 제어 로직(Orchestration Logic)**을 중심으로 설계됩니다.

#### 1. 시스템 상태 정의 (State Definition)
시스템은 항상 다음 세 가지 핵심 상태 변수를 관리해야 합니다.

*   `data_integrity_status`: 데이터 수신 및 검증의 현재 상태 (`'OK'`, `'FAIL'`, `'PENDING'`)
*   `risk_hedge_active`: 리스크 헷지 모듈이 현재 활성화되어 있는 상태 (`True`/`False`)
*   `system_safe_mode`: 전체 시스템이 안전 모드에 진입했는지 여부 (`True`/`False`)

#### 2. 핵심 알고리즘 흐름 (Core Algorithm Flow)

데이터 스트림에서 데이터가 수신될 때마다 다음의 순서로 로직을 실행합니다.

1.  **데이터 수신 (Data In):** 키움 RestApi 등으로부터 실시간 주식 데이터를 수신합니다.
2.  **무결성 검증 (Integrity Check):** 데이터 무결성 검증 모듈(F1 모듈 기반)이 수신된 데이터의 정확성, 일관성, 시간 동기화 등을 검사합니다.
3.  **상태 업데이트 (State Update):** 검증 결과에 따라 `data_integrity_status`를 업데이트합니다.
    *   만약 검증 실패 시: `data_integrity_status = 'FAIL'`
4.  **안전장치 판단 (Safety Decision):** 시스템은 `data_integrity_status`와 사전 정의된 KPI($MDD_{limit}$, $SR_{target}$)를 비교하여 다음을 결정합니다.

    *   **Case A: 데이터 무결성 실패 시 (Trigger)**
        *   만약 `data_integrity_status == 'FAIL'` 이면, 즉시 **Hard Stop**을 실행합니다.
        *   `system_safe_mode = True`로 설정합니다.
        *   모든 자동매매 및 예측 알고리즘의 실행을 중단(Halt)합니다.
        *   리스크 헷지 모듈은 강제로 가장 보수적인 조치(**Case B: 손실 제한**)를 활성화하여 포지션 청산 또는 최대 손실 도달 시 즉시 정지 명령을 내립니다.

    *   **Case B: 정상 작동 시 (Normal Operation)**
        *   만약 `data_integrity_status == 'OK'` 이면, 시스템은 정상적인 예측 및 자동매매 로직을 계속 수행합니다. 리스크 헷지 모듈은 설정된 KPI에 따라 동적으로 조정됩니다.

5.  **결과 보고 (Reporting):** 모든 상태 변경 및 안전장치 실행 결과를 로그 파일(`system_log.txt` 등)에 기록하여 감사 추적(Audit Trail)을 확보합니다.

#### 3. 예외 처리 전략 (Exception Handling Strategy)

통합 과정에서 발생할 수 있는 주요 예외 상황과 그 처리 방안은 다음과 같습니다.

| 발생 상황 | 원인 | 시스템 반응 (Action) | 담당 모듈 |
| :--- | :--- | :--- | :--- |
| **데이터 불일치** | RestApi 데이터 누락, 이상치 발견 등 (`FAIL`) | 즉시 `system_safe_mode = True` 설정 및 모든 알고리즘 정지. Case B 강제 실행. | 중앙 제어 로직 |
| **검증 모듈 오류** | 무결성 검증 자체에서 예기치 않은 에러 발생 (Code Exception) | 로그 기록 후, 시스템을 '경고 상태(Warning State)'로 전환하고, 자동매매는 일시 중단(Pause). | 데이터 무결성 모듈 |
| **모듈 통신 오류** | 리스크 헷지 모듈과의 API/함수 호출 실패 | 즉시 `system_safe_mode = True` 설정. (최악의 경우 강제 종료) | 중앙 제어 로직 |
