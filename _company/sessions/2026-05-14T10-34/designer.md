# 🎨 Designer — 데이터 기반의 신뢰감과 시스템 안정성을 시각적으로 표현하는 채널의 핵심 컬러 팔레트, 타이포그래피 스타일 및 썸네일 디자인 가이드라인 초안을 리스크 관리 모듈의 성공 사례를 포함하여 재정립하라.

🎨 Designer: 작업 시작합니다.

CEO 지시에 따라, 리스크 관리 모듈의 성공 사례를 기반으로 '데이터 기반 신뢰감과 시스템 안정성'을 시각적으로 표현하는 채널의 핵심 디자인 가이드라인을 재정립하겠습니다.

이 가이드라인은 **'시스템의 안정성을 보장하는 엔지니어링 포지션'**을 시청자에게 전달하는 데 중점을 둡니다.

## 💡 브랜드 비주얼 시스템 재정립: 갓더주식 (GodTheStock)

### 1. 핵심 컬러 팔레트 (Core Color Palette)

시스템 안정성과 데이터 기반 신뢰를 표현하기 위해, 기존의 'Deep Navy Blue'와 'System Green'을 주축으로 하고, 리스크 관리의 중요성을 강조하는 보조 색상을 추가합니다.

| 역할 | 색상 이름 | HEX 코드 | 사용 목적 및 의미 |
| :--- | :--- | :--- | :--- |
| **Primary (신뢰/안정)** | Deep Navy Blue | `#0A1931` | 시스템의 깊이, 전문성, 신뢰감. 채널의 메인 배경 및 헤더에 사용. |
| **Secondary (성장/안전)** | System Green | `#10B981` | 긍정적인 결과(수익)와 안전한 성장(Risk-Managed Growth). 핵심 CTA 및 성공 지표 강조. |
| **Accent (데이터/분석)** | Teal / Cyan | `#2DD4BF` | 데이터 시각화, 그래프 라인, 기술적 분석 영역. 차분하고 분석적인 느낌 부여. |
| **Warning (리스크)** | Amber / Red | `#F59E0B` / `#EF4444` | 시스템 안전장치 작동(System Safe Mode), 손실 제한($MDD_{limit}$) 경고, 위험 신호에만 사용. |
| **Background (배경)** | Off-White / Light Gray | `#F7F8FC` | 텍스트 가독성을 극대화하고 데이터 영역을 깔끔하게 분리. |

**🎨 컬러 활용 원칙:**
1.  **Deep Navy Blue**를 주 배경으로 사용하여 전문적인 분위기를 확립합니다.
2.  **System Green**을 성공 지표나 시스템이 안전할 때의 상태(Safe Mode)에만 사용하며, 과도한 사용은 지양합니다.
3.  **Teal/Cyan**을 데이터 시각화(차트, 그래프)에 사용하여 분석적인 느낌을 강화합니다.

---

### 2. 타이포그래피 스타일 (Typography Style)

데이터 기반의 명확성과 전문성을 극대화하기 위해, 가독성이 높고 모던하며 구조적인 산세리프 계열을 사용합니다.

*   **메인 폰트 (Headlines & Display):** **Inter** 또는 **Poppins**
    *   **선택 이유:** 현대적이고 깔끔하며, 숫자와 데이터 시각화에 최적화된 명확한 형태를 가집니다.
*   **본문 폰트 (Body & Data Labels):** **Roboto** 또는 **Pretendard**
    *   **선택 이유:** 복잡한 수치와 설명(리스크 모듈 설명)을 읽는 데 있어 높은 명료성을 제공하여 데이터의 신뢰도를 높입니다.

**✍️ 타이포그래피 활용 원칙:**
1.  **계층 구조 강조:** 제목은 굵고 크게, 데이터 값과 핵심 메시지는 **Teal/Cyan**으로 강조하여 시각적 계층을 명확히 합니다.
2.  **숫자의 중요성:** 모든 중요한 수치($MDD_{limit}$, $SR_{target}$)는 별도의 박스나 색상(Warning/System Green)으로 분리하여, 숫자가 곧 시스템의 상태임을 각인시킵니다.

---

### 3. 썸네일 디자인 가이드라인 (Thumbnail Design Guidelines)

썸네일은 '과장된 예측'이 아닌 **'시스템의 안정성 보고서'**라는 인상을 주어야 합니다. 데이터 기반 신뢰를 시각화하는 데 중점을 둡니다.

#### **A. 레이아웃 컨셉: 대시보드 스타일 (Dashboard Style)**

*   **기본 구조:** 좌측 또는 우측에 핵심 메시지/결과, 중앙에 시스템 상태 및 데이터 시각화를 배치합니다.
*   **배경:** Deep Navy Blue 계열을 사용하여 전문성을 유지합니다.
*   **핵심 요소:** 그래프나 차트 이미지를 반드시 포함하여 '데이터 기반'임을 증명해야 합니다.

#### **B. 핵심 디자인 구성 (3안 비교)**

| 컨셉 | 제목/메시지 강조 | 시각적 특징 | 적합한 콘텐츠 유형 |
| :--- | :--- | :--- | :--- |
| **컨셉 1: 시스템 안정성 보고서** | "손실 통제 시스템의 비밀" (가장 높은 신뢰도) | 차트(Teal/Cyan)와 핵심 KPI($MDD_{limit}$)를 중심으로, 중앙에 녹색으로 '안정' 상태 표시. | 리스크 관리 모듈 작동 원리 설명, 시스템 안정성 강조 영상. |
| **컨셉 2: 예측 vs 통제** | "예측 실패? 시스템이 지킨다" (대립 구도) | 좌우 분할 레이아웃. 왼쪽은 불안정한 빨간색/주황색(예측 실패), 오른쪽은 안정적인 녹색(시스템 통제). | 시장 변동성과 시스템의 대응 능력을 비교하는 영상. |
| **컨셉 3: 데이터 기반 증명** | "수익률보다 중요한 것은 리스크 관리" (철학 강조) | 깔끔한 타이포그래피 중심. 배경에 은은하게 네트워크/코드 패턴을 삽입하고, 핵심 메시지를 큰 글씨로 중앙에 배치. | 브랜드 철학 소개, 채널의 신뢰도 구축 영상. |

#### **C. 썸네일 적용 예시 (컨셉 1 상세)**

*   **배경:** Deep Navy Blue (`#0A1931`)
*   **중앙 요소:** 대형 차트 또는 시스템 상태 표시 박스 (Teal/Cyan 배경)
*   **텍스트 배치:**
    *   상단: **시스템 안정성 보고서** (White, Bold)
    *   하단 강조: **$MDD_{limit}$이 당신의 손실을 통제한다.** (System Green으로 강조)
