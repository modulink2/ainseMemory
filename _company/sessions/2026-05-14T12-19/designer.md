## 🛡️ 갓더주식 비주얼 시스템 가이드라인 (Visual System Guide)

핵심 정체성인 **'데이터 기반 신뢰(Deep Navy Blue)'**와 **'시스템 안정성 및 성장(System Green)'**을 중심으로 디자인 요소를 정의합니다.

### 1. 컬러 팔레트 (Color Palette)

| 역할 | 색상명 | HEX 코드 | 용도 및 의미 |
| :--- | :--- | :--- | :--- |
| **Primary (신뢰)** | Deep Navy Blue | `#0A1931` | 시스템의 안정성, 전문성, 데이터 기반 신뢰를 상징. 메인 배경 및 헤더 사용. |
| **Secondary (안정/액션)** | System Green | `#16A34A` | 시스템의 안전장치(Shield)와 긍정적인 성장(Hedger)을 상징. CTA 버튼, 성공 지표, 핵심 데이터 강조에 사용. |
| **Accent (강조)** | Light Gray / White | `#F8F9FA` | 배경 및 정보 영역으로 사용하여 Navy Blue와 Green의 대비를 극대화하고 가독성을 확보. |
| **Warning (경고)** | Amber/Red | `#F59E0B` / `#EF4444` | 리스크 경고, 손실 임박 상황 등 시스템 알림에만 제한적으로 사용. |

### 2. 타이포그래피 (Typography)

**목표:** 데이터 기반의 전문성과 명확성을 전달합니다.

*   **헤드라인/핵심 지표:** **Inter Bold** 또는 **Noto Sans KR Bold** (데이터 강조를 위해 간결하고 명료함).
*   **본문/설명:** **Pretendard** 또는 **Noto Sans KR Regular** (높은 가독성을 확보하여 복잡한 리스크 정보를 쉽게 이해하도록 함).

### 3. 레이아웃 및 컴포넌트 스타일 (Layout & Component Style)

| 컴포넌트 | 기본 스타일 적용 | Basic Shield (진입 장벽) | Pro Hedger (핵심 수익) |
| :--- | :--- | :--- | :--- |
| **배경** | Deep Navy Blue (`#0A1931`)를 주 색상으로 설정. | 안정적인 배경 유지. | 안정성을 강조하기 위해 미세하게 어두운 톤 조정 가능. |
| **핵심 지표** | System Green (`#16A34A`)을 사용하여 모든 성공/안정성 지표(MDD, SR)를 표시. | 녹색은 최소한의 안전 기준만 표시. | 녹색을 통해 MDD/SR 개선 추이를 역동적으로 강조. |
| **구분선** | 얇고 명확한 Light Gray 선 사용. | 단순하고 명료하게 구분. | 복잡한 기능 구분을 위한 세밀한 레이아웃 적용. |
| **CTA 버튼** | System Green 배경 (`#16A34A`), White 텍스트. | '시작하기' (Basic Shield 활성화) | '최적화하기' (Pro Hedger 실행) |

## 🖼️ 썸네일 디자인 초안 (Thumbnail Design Drafts)

핵심 메시지인 **"시스템 안정성과 리스크 관리"**를 중심으로 Basic과 Pro 티어를 시각적으로 차별화합니다. 모든 썸네일은 Deep Navy Blue 배경 위에 System Green을 포인트로 사용하여 신뢰감을 극대화합니다.

### 1. Basic Shield 썸네일 컨셉 (진입 장벽 & 안정성 강조)

**핵심 메시지:** "손실 방지" 및 "최소한의 안전장치 확보"

| 컨셉 | 시각적 설명 | 레이아웃 좌표/요소 배치 |
| :--- | :--- | :--- |
| **안정성 강조형 (추천)** | 중앙에 굵은 녹색 프레임 안에 'STOP-LOSS' 아이콘과 함께, 파란색 배경 위에 시스템의 견고한 방패(Shield) 이미지를 배치. | 좌측 상단: Deep Navy Blue 배경 / 중앙: 큰 System Green Shield 로고 / 하단: "Basic Shield: 손실 방지 시작" (흰색 폰트). |
| **진입 장벽 강조형** | 시스템의 '시작점'을 시각적으로 표현. 단순한 UI 화면과 함께 제한적인 안전선(Boundary Line)을 강조. | 좌측: Navy Blue 배경 / 중앙: 단순화된 차트/UI 화면 / 우측: 녹색으로 표시된 최소 안전 범위 (Minimal Safety Zone). |
| **데이터 기반형** | 데이터의 '기준점'에 초점을 맞춤. 정적인 그래프와 함께 명확한 경계선을 제시. | 전체 배경을 Navy Blue로 채우고, 중앙에 System Green으로 표시된 현재 포트폴리오의 '최소 안전 마진' 수치 강조. |

### 2. Pro Hedger 썸네일 컨셉 (수익 극대화 & 능동적 관리 강조)

**핵심 메시지:** "리스크 헷지" 및 "수익 최적화"

| 컨셉 | 시각적 설명 | 레이아웃 좌표/요소 배치 |
| :--- | :--- | :--- |
| **최적화 강조형 (추천)** | 역동적인 녹색 흐름(Flow)을 사용하여 리스크가 능동적으로 관리되고 있음을 표현. 복잡한 알고리즘의 안정적인 통합을 시각화. | 좌측: Navy Blue 배경 / 중앙: 시스템 Green으로 연결된 추세선 및 헷지 로직 다이어그램 / 우측: "Pro Hedger: MDD/SR 기반 최적화" (흰색 폰트). |
| **성장 강조형** | 리스크 관리가 곧 수익 극대화로 이어진다는 점을 역동적으로 표현. 상승하는 녹색 곡선과 안정적인 시스템 프레임을 결합. | 상단: 큰 System Green 성장 그래프 / 중앙: '헷지'와 '수익'의 연결고리 강조 / 하단: "Pro Hedger: 수익 목표 달성 로직" (흰색 폰트). |
| **통제권 강조형** | 사용자가 시스템을 완전히 통제하고 있음을 암시. 복잡한 데이터가 명확하게 제어되는 느낌. | 중앙에 대시보드 레이아웃의 핵심 부분(MDD/SR 지표)을 녹색으로 하이라이트하고, 주변을 Navy Blue로 감싸는 형태. |
