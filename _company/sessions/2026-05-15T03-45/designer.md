## 1. 브랜드 아이덴티티 및 컬러 시스템
### 1.1. 핵심 색상 팔레트
| 역할 | 색상 이름 | HEX 코드 | 용도 |
| :--- | :--- | :--- | :--- |
| **Primary (신뢰)** | Deep Navy Blue | `#0A1931` | 배경, 주요 섹션, 네비게이션 바 |
| **Accent (안전)** | System Green | `#00C853` | 긍정적 지표, 안전성 레벨 강조 (PTI > 기준) |
| **Secondary (경고)** | Amber/Orange | `#FFB300` | 주의 필요 지표, 리스크 경고 (PTI 위험 구간) |
| **Neutral** | Light Gray | `#F5F5F5` | 카드 배경, 보조 정보 영역 |
| **Text** | White | `#FFFFFF` | 텍스트 및 주요 데이터 표시 |

### 1.2. 타이포그래피 시스템
*   **폰트:** Pretendard (또는 시스템 기본 Sans-serif)
*   **Hierarchy:** Data-Driven, 명확성 최우선. 숫자는 가장 크게 강조하며, 설명은 간결하게 배치.

### 1.3. 데이터 시각화 원칙 (PTI 중심)
*   **핵심 지표 우선순위:** PTI(시스템 안전성 지수) > ROI(수익률) > 예측 점수.
*   **안전성 레벨 시각화:** '안전성 레벨'은 가장 중요한 KPI로, **System Green (매우 안전)**, **Amber (주의)**, **Red (위험)** 세 단계로 명확하게 구분하여 표시한다.

## 2. 대시보드 레이아웃 및 컴포넌트
### 2.1. 전체 구조 (DashboardLayout 기반)
*   **레이아웃:** 사이드바(Navigation) + 상단 바(Header) + 메인 콘텐츠 영역 통합 (DashboardLayout 적용).
*   **사이드바:** Deep Navy Blue 배경으로, 주요 메뉴는 System Green 하이라이트로 표시.
*   **상단 헤더:** 현재 시스템 상태(전체 리스크 요약)를 간결하게 표시.

### 2.2. 핵심 컴포넌트 디자인 상세 (Basic/Pro 티어 비교)
| 컴포넌트 | Basic Shield (안정성 중시) | Pro Hedger (수익 및 안전 균형) | 시각화 강조점 |
| :--- | :--- | :--- | :--- |
| **PTI 지표** | 시스템 안정성 지수(PTI)를 가장 크게 표시. 녹색 톤으로 '안전'을 최우선 강조. | PTI와 ROI를 병렬로 제시하며, PTI가 ROI보다 우선함을 명시. | 안전성 레벨 (Green/Amber/Red 바) |
| **KPI 카드** | 최소화된 정보, PTI 값과 리스크 상태만 표시. | PTI, ROI, 헤징 효율성을 모두 포함하여 다차원적 분석 제공. | 데이터 밀도(Data Density) |
| **리스크 시각화** | 단순한 안전성 레벨 바 (Green/Red). | 복합적인 리스크 매트릭스 (PTI-ROI 관계 그래프) 삽입. | 관계 파악 (Correlation) |

### 2.3. 최종 디자인 가이드라인 요약
1.  **색상 사용:** Deep Navy Blue를 주조색으로 사용하여 전문성과 안정감을 확보하고, System Green을 통해 '안전'과 '성공'의 경로를 명확히 제시한다.
2.  **데이터 우선순위:** 모든 시각화는 **PTI(시스템 안전성)**가 **ROI(수익률)**보다 우선한다는 점을 데이터 구조와 색상으로 반영해야 한다.
3.  **사용자 경험 (UX):** 복잡한 수치 대신, 사용자가 한눈에 자신의 시스템 안정 상태를 파악하고 즉각적인 조치를 취할 수 있도록 '안전성 레벨' 시각화 로직을 가장 상단에 배치한다.
