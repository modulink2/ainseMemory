# 📄 비즈니스 요구사항 명세서: PTI/ROI 연관성 기반 리스크 최소화 전략

**문서 버전:** 1.0
**작성 일자:** 2026-05-16
**대상:** 개발팀, 데이터 분석팀
**목표:** 시스템 안정성(PTI)과 수익률(ROI)의 수학적 연관성을 극대화하여 예측 알고리즘 기반의 변동폭(MDD)을 최소화하는 구체적인 비즈니스 목표 및 KPI 측정 기준 정의.

## 1. 핵심 원칙 (Guiding Principle)

**"안정성이 곧 수익"**: 시스템 안정성 지수(PTI)가 높을수록 예측 알고리즘의 결과에 대한 신뢰도가 높아지며, 이는 궁극적으로 최대 손실 위험(MDD)을 최소화하는 직접적인 수단이 된다.

## 2. 핵심 KPI 정의 (Key Performance Indicators)

개발팀은 다음 세 가지 지표를 실시간으로 계산하고 대시보드에 표시해야 한다.

### 2.1. 시스템 안정성 지수 (PTI: Platform Stability Index)
**정의:** 예측 알고리즘의 입력 데이터 무결성, API 통신 성공률, 내부 로직 오류 발생 빈도 등을 종합하여 시스템이 정상적으로 작동하고 예측을 수행할 수 있는 상태를 나타내는 지표.

*   **계산 요소 (예시):** API 응답 성공률(%), 예측 모듈 실행 오류율(%), 데이터 무결성 검증 실패 횟수, 실시간 데이터 입력 지연 시간.
*   **목표:** $\text{PTI} \ge 80$ (최소 안정 기준)

### 2.2. 수익률 변동폭 (MDD: Maximum Drawdown)
**정의:** 특정 기간(예: 최근 3개월 또는 1년) 동안 발생한 최고 자산 가치 대비 최저 자산 가치 사이의 최대 손실률. 이는 알고리즘 예측 결과가 실제 시장 상황에 얼마나 민감하게 반응했는지 측정한다.

*   **목표:** $\text{MDD} \le X\%$ (수익 목표에 따라 설정)

### 2.3. 안정성-수익 연관 지수 ($\text{PTI/ROI Ratio}$)
**정의:** 시스템 안정성과 수익률 간의 관계를 정량화하는 핵심 지표. 이 비율이 높을수록(안정성이 높고 수익도 좋음) 이상적인 상태로 간주한다.

*   **계산식 (제안):** $\text{Ratio} = \frac{\text{PTI}}{\text{ROI}_{\text{Rolling Window}}}$

## 3. 비즈니스 목표 및 측정 기준 (Business Goals & Measurement Criteria)

개발팀은 다음의 구체적인 목표를 달성하기 위한 시스템 로직을 설계해야 한다.

### 3.1. 최우선 목표: MDD 최소화 전략
**목표:** $\text{PTI} \ge 80$ 상태에서, $\text{MDD}$를 시장 평균 대비 $Y\%$ 이하로 유지하는 것을 최우선으로 설정한다.

*   **KPI 1 (안정성 보장):** $\text{PTI}_{\text{Target}} = 90$ (최상위 안정 목표)
*   **KPI 2 (리스크 제어):** $\text{MDD}_{\text{Limit}} = 15\%$ (시스템이 허용 가능한 최대 손실 한계)

### 3.2. 동적 리스크 관리 로직 구현
PTI가 특정 임계치 이하로 떨어질 경우, 시스템은 자동적으로 경고 및 제어 모드로 전환해야 한다.

*   **경고 조건:** $\text{PTI} < 80$ 일 때, 즉시 사용자에게 '시스템 안정성 위험' 알림을 발생시킨다.
*   **자동 조치 (제어 모드):** $\text{PTI} < 80$ 상태에서, 알고리즘의 자동 매매 실행을 **일시 중지(Pause)**하고, 수동 검토를 요구한다.

### 3.3. ROI 연관성 극대화
알고리즘이 예측한 수익률($\text{ROI}_{\text{Predicted}}$)과 실제 달성된 수익률($\text{ROI}_{\text{Actual}}$)의 차이를 분석하여, PTI가 높을 때 예측 정확도가 실제 수익으로 얼마나 잘 연결되는지 검증한다.

*   **검증 지표:** $\text{Correlation}(\text{PTI}, \text{ROI}_{\text{Actual}})$이 $+0.8$ 이상일 경우, 알고리즘의 신뢰도를 최고 수준으로 인정한다.

## 4. 개발팀 전달 사항 (Action Items for Development Team)

1.  **데이터 추적 모듈 개발:** PTI와 MDD를 실시간으로 계산하고 기록하는 백엔드 모듈을 구현해야 한다.
2.  **동적 제어 로직 통합:** $\text{PTI}$에 따라 알고리즘의 실행 권한(자동매매 활성화/비활성화)을 제어하는 안전장치 로직을 시스템 코어에 통합한다.
3.  **KPI 대시보드 구축:** 사용자가 PTI, ROI, MDD, 그리고 $\text{PTI/ROI Ratio}$를 한눈에 파악할 수 있는 시각화된 대시보드를 구현한다.
