### 1.1. 최종 목표 재확인
*   **핵심 철학:** 시스템 안정성(PTI)이 곧 수익(ROI).
*   **디자인 방향:** 데이터 기반 신뢰 (Deep Navy Blue & System Green).

## 2. 디자인 가이드라인 (Pre-emptive Design Framework)

개발 완료 전, 데이터가 들어갈 영역에 대한 **구조적 요구사항**을 정의하여 개발팀과의 싱크를 맞춥니다.

### 2.1. 핵심 UI 컴포넌트 구조 정의
최종 대시보드는 '안정성 중심'으로 설계되어야 하며, 다음 세 가지 주요 섹션과 데이터 흐름을 포함합니다.

| 섹션 | 목적 | 핵심 지표 (KPI) | 디자인 강조 요소 |
| :--- | :--- | :--- | :--- |
| **A. 시스템 안정성 대시보드 (PTI Focus)** | 플랫폼의 신뢰도와 안전성을 최우선으로 표시 | **PTI (System Stability Index)**, 리스크 헷지 상태 | Deep Navy Blue 배경, System Green 성공/경고 표시 |
| **B. 예측 성과 분석 (ROI Focus)** | 알고리즘의 실제 수익률 및 예측 정확도를 측정 | **ROI (Return on Investment)**, 예측 성공률 (%) | ROI 변화율(Delta) 강조, 안정성 대비 수익 관계 시각화 |
| **C. 실시간 포트폴리오 (Action Focus)** | 현재 보유 자산과 자동매매 상태를 즉시 확인 | 현재 강성주 지수, 자동매매 활성화 여부 | 명확한 CTA 버튼 (안정성 기반), 실시간 데이터 위젯 |

### 2.2. 컬러 및 타이포그래피 최종 점검
*   **Primary Color:** Deep Navy Blue (`#0A1931`) - 안정성과 신뢰성 표현.
*   **Accent Color:** System Green (`#33FF99`) - 시스템 안전 상태, 성공적인 리스크 헷지.
*   **Warning Color:** Amber/Red (시스템 불안정 시) - PTI 임계값 미달 또는 위험 경고 시 사용.
*   **Typography:** 산세리프 계열(예: Pretendard 또는 Noto Sans KR)을 사용하여 데이터의 명확성과 가독성을 극대화합니다.
